## 2026 VAST 可视化分析挑战赛 参赛提交须知

### 一、截止时间

提交截止：2026 年 7 月 17 日 太平洋东部时区晚 11:59
参赛队伍可提交多份参赛作品，但每份作品都必须完整包含下文规定的全部材料。

### 二、关于 GitHub 提交作品的说明
近年越来越多队伍通过 GitHub 提交作品，并附上在线演示链接，这一方式我们十分认可。但请务必遵守以下要求：

1. 提交作品时必须打上 GitHub 标签（tag）或发布正式版本（release）；
2. 完整的参赛信息表必须在截止时间前全部提交，不允许仅在 GitHub 内放置信息表链接；
3. 作品一经提交，不得自行修改更新，仅当 VAST 挑战赛组委会专门提出要求时才可调整。

### 三、完整提交包需在 2026 年 7 月 17 日前完成提交，包含一份填写完整的总结表格，以及一段时长 4 分钟、演示可视化分析软件运行效果的讲解视频。
#### （一）总结表格填写规范
1. 将表格文件重命名为index.htm，文件后缀必须保留.htm。可使用 Word 等文字编辑软件填写内容；
2. 作品命名规则：机构简称 - 联系人姓氏 - 挑战赛代号。
- 示例：马里兰大学 Jones 博士团队参加迷你挑战赛 2，文件命名为UMD-Jones-MC2；
- 若提交多份作品，在文件名末尾添加数字区分，如UMD-Jones-MC2-1；
3. 列出全部团队成员、所属单位、电子邮箱，指定一名主联系人，该人员需负责接收赛事相关问询与通知；
4. 标注是否为学生团队：学生团队定义为由学生牵头、全部开发工作由学生完成，课程项目类参赛作品一般归为此类；
5. 列明解决方案开发过程中使用的全部分析工具：
- Tableau、SAS 等商用工具直接写名称即可；
- 非商用自研工具需提供：工具名、开发方（个人 / 公司）、工具访问链接，补充开发背景说明（例：SPINWIZ 由西米德兰兹大学 2011 春季 CS459 可视化课程 Smith 教授开发，本参赛团队适配改造用于本次挑战赛）；
6. 预估全队完成本份作品投入的总工时；多份作品需分开填写各自工时；
7. 授权声明：是否允许组委会将你的参赛作品公开发布至《可视化分析基准数据库》；
8. 附带带语音解说的讲解视频：
- 时长不得超过 4 分钟，完整讲解你的整套方案；
- 视频格式要求.wmv；
- 文件体积允许的情况下，将视频打包进电子提交包；若文件过大，上传至外网并提供有效访问链接；
- 全程保证链接在赛事评审周期内可正常访问（多位评委将分时段下载查看）；
- 若无视频托管平台，可发邮件联系vast_challenge@ieeevis.org沟通替代方案；
- 赛事方会留存视频存档，后续需单独提供视频原文件。

#### （二）视频制作参考要点
- 清晰说明你自研 / 选用的可视化分析软件如何支撑本次数据分析；
- 明确本次分析的起点，以及该起点的筛选依据；
- 解释核心可视化设计决策背后的思路；
- 说明可视化视图、交互功能对数据分析起到的作用；
- 若使用筛选、数据变换功能，说明所用工具与操作执行顺序；
- 视图中若展示异常数据，说明如何将异常纳入分析流程；
- 对文中专业术语（如 “不确定性”）给出清晰定义；
- 说明方案中所有关键假设；
- 完整作答本次迷你挑战赛配套的指定问题（问题已附在总结表格内）。

### 四、作品打包与提交流程
1. 将总结表格命名为index.htm保存在本地；可直接在表格内填写作答内容，或通过超链接关联外部补充文件。
- 补充说明：通过 GitHub 提交的作品，额外需要上传一份表格 PDF 版。
2. 在本地新建文件夹，文件夹名与你的作品命名保持一致；将填好的index.htm、设计图、可选视频、故事板、其他附图全部放入文件夹；index.htm内所有文件链接需使用相对路径，保证文件夹迁移后链接仍可正常打开。
3. 将整个文件夹压缩为 ZIP 压缩包，压缩包名称和作品名统一（例：UMD-Jones-MC2.zip）；
- 若压缩包超过 50MB：视频等大文件单独上传外网，仅将小文件打包进 ZIP；
- 系统仅支持≤50MB 的 ZIP 文件，rar、tar 等其他压缩格式不予接收。
4. 2026 IEEE VAST 挑战赛使用 Precision 会议系统（PCS）处理提交与评审：
- 系统地址：https://new.precisionconference.com/~vgtc/
- 无账号需先注册，注册入口在登录页；
- 登录后点击「Submissions（提交）」，依次选择：协会 VGTC、会议 / 期刊 VIS 2026、赛道 VIS 2026 VAST 挑战赛。
